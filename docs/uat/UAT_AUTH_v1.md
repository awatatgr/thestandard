# UAT: 認証フロー (Authentication Flow)

**バージョン**: v1.0
**作成日**: 2026-03-19
**テスト範囲**: マジックリンクログイン、認証ガード、ログアウト、セッション管理、ロールベースアクセス制御
**優先度フォーカス**: P0-P1
**準拠規格**: ISTQB UAT ガイドライン / IEEE 829

---

## 1. ユースケース (User Stories)

### US-01: マジックリンクでログインする (P0)

> 閲覧者として、メールアドレスだけでログインできるようにしたい。パスワードを覚える必要がないからだ。

**受入基準**:
- AC-01: `/login` でメールアドレスを入力し「ログインリンクを送信」をクリックすると、OTPメールが送信される
- AC-02: 送信後「メールを確認してください」画面が表示され、入力したメールアドレスが確認できる
- AC-03: メール内のマジックリンクをクリックすると、`/` (動画一覧) にログイン状態で遷移する
- AC-04: 無効なメールアドレスの場合、HTML5バリデーションでフォーム送信がブロックされる
- AC-05: API エラー時は赤色エラーボックスにメッセージが表示される

### US-02: 未認証ユーザーを保護ページからブロックする (P0)

> プロダクトオーナーとして、未認証ユーザーが動画コンテンツにアクセスできないようにしたい。コンテンツの価値を保護するためだ。

**受入基準**:
- AC-06: 未認証で `/` にアクセスすると `/login` にリダイレクトされる
- AC-07: 未認証で `/videos/:id` にアクセスすると `/login` にリダイレクトされる
- AC-08: `/embed/:id` は認証不要でアクセスできる
- AC-09: `/login`, `/signup`, `/open-source` は認証不要でアクセスできる

### US-03: ログアウトする (P0)

> 閲覧者として、他人のデバイスを使った後にログアウトできるようにしたい。アカウントを安全に保つためだ。

**受入基準**:
- AC-10: ヘッダーのユーザーメニューから「ログアウト」をクリックするとセッションが破棄される
- AC-11: ログアウト後、保護ページにアクセスすると `/login` にリダイレクトされる

### US-04: ロールに応じたアクセス制御 (P0)

> プロダクトオーナーとして、管理画面は管理者のみアクセスできるようにしたい。誤操作を防ぐためだ。

**受入基準**:
- AC-12: admin ロールのユーザーはヘッダーメニューに「管理画面」リンクが表示される
- AC-13: admin ロールのユーザーは `/admin` 以下にアクセスできる
- AC-14: 閲覧者ロールで `/admin` にアクセスすると `/` にリダイレクトされる
- AC-15: admin ロールのユーザーにはメニュー内に「Admin」バッジが表示される

### US-05: セッションの永続化と復元 (P1)

> 閲覧者として、ブラウザを閉じて再度開いたときにログイン状態が維持されていてほしい。毎回ログインし直すのは手間だからだ。

**受入基準**:
- AC-16: ログイン後、同一タブ内でページ遷移してもセッションが維持される
- AC-17: ブラウザタブを閉じて再度開いたとき、Supabase セッションが有効ならログイン状態が復元される
- AC-18: セッショントークンが期限切れの場合、`/login` にリダイレクトされる

### US-06: ログイン済みユーザーのログインページ遷移 (P1)

> 閲覧者として、既にログインしている状態で `/login` にアクセスしたとき、自動的にトップページに転送されたい。無駄なステップを省くためだ。

**受入基準**:
- AC-19: ログイン済みで `/login` にアクセスすると `/` にリダイレクトされる
- AC-20: `/signup` にアクセスすると `/login` にリダイレクトされる（マジックリンクで統一）

### 前提条件チェックリスト

テスト実行前に以下を確認:

- [ ] `VITE_SUPABASE_URL` と `VITE_SUPABASE_ANON_KEY` が設定済み (認証有効)
- [ ] テスト用アカウント3件が Supabase に登録済み
- [ ] `coach@example.com` の `profiles.role` が `admin` に設定済み
- [ ] `admin@example.com` と `viewer@example.com` の `profiles.role` が `admin` 以外 (または未設定)
- [ ] テスト環境 (`https://thestandard.pages.dev` または `http://localhost:8080`) にアクセス可能
- [ ] 動画データが少なくとも1件公開済み (embed テスト用)
