# THE STANDARD — UAT (User Acceptance Test)

## 概要

THE STANDARD のユーザー受入テスト。プロダクトが実際のユーザーシナリオで正しく動作することを検証する。

## ペルソナ

| ペルソナ | 説明 | 対象ファイル |
|---------|------|------------|
| 視聴者 (Viewer) | ログインして動画を視聴するユーザー。マルチアングルでフォーム確認 | `a-viewer.md` |
| 管理者 (Admin) | 動画の登録・公開・管理を行うプロデューサー | `b-admin.md` |

## テスト環境

| 項目 | 値 |
|------|------|
| URL | `https://thestandard.pages.dev` (本番) / `http://localhost:8080` (開発) |
| ブラウザ | Chrome 最新版 |
| モバイル | iPhone 14 / Safari |
| 認証 | Supabase マジックリンク |

## テスト実行ガイド

### 記号の読み方

- `[ ]` — 未検証の期待結果（チェックリスト）
- **前提条件** — テスト開始前に満たすべき状態
- **手順** — 番号順に実行するステップ
- **期待結果** — 各ステップ後に確認するチェック項目
- **エッジケース** — 追加で確認すべき境界条件

### Playwright E2E テストとの対応

各UATケースに、自動テスト（`e2e/*.spec.ts`）でカバー済みかどうかを記載。
`[自動]` = Playwright でカバー済み、`[手動]` = 手動テストが必要。

### テスト実行手順

```bash
# 自動テスト（Playwright）
npm run test:e2e

# 手動テスト
# 1. npm run dev でローカルサーバー起動
# 2. 各 UAT ケースの手順に従って検証
# 3. チェックリストにチェックを入れる
```
