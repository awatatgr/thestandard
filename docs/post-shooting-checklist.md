## 当日（撮影直後）

### Step 1: データ転送
```bash
# SSD2台をPCに接続して実行
./scripts/ingest.sh
```
- A/Bカメを自動判別して `~/footage/2026-03-16/` にコピー
- 判別できない場合は手動で指定

### Step 2: バックアップ
- [ ] 別のドライブにコピーを作る（最低2箇所保存）
```bash
cp -r ~/footage/2026-03-16 /path/to/backup/2026-03-16
```
- [ ] バックアップ完了を確認してからSSDのデータを消す（消すのは次の撮影直前でOK）

### Step 3: 撮影ログ整理
- [ ] `~/footage/2026-03-16/shooting-log.md` を開く
- [ ] 各テイクのOK/NGを記入（記憶が新しいうちに）
- [ ] 使わないテイクにNGマーク
- [ ] 特に良かったテイクにメモ

## 翌日以降（編集 → 公開）

### Step 4: DaVinci Resolve 処理
- [ ] DaVinci Resolve を起動
- [ ] 新規プロジェクト作成:「TheStandard_2026-03-16_Yoga」
- [ ] メディアプールに A-cam / B-cam フォルダをインポート

#### 同期
- [ ] A/Bカメ素材を選択 → 右クリック → 「オーディオに基づいて自動同期」
- [ ] 同期結果を再生して確認（ずれていたら手動調整）

#### カラーグレーディング
- [ ] カラーページに移動
- [ ] 全クリップを選択
- [ ] ノードに LUT 適用:「Apple Log to Rec.709」
- [ ] 明るさ・コントラストの微調整（必要に応じて）

#### カット編集（種目ごと）
- [ ] 種目ごとにタイムラインを作成（例: `01_downdog`, `02_updog`...）
- [ ] NG部分・無駄な間をカット
- [ ] 導入 → NG例 → 正解 → 質問 → バリエーション の流れに整理
- [ ] 各種目 3-5分以内に収める

#### エンコード
- [ ] デリバーページに移動
- [ ] 設定:
  - フォーマット: MP4
  - コーデック: H.264
  - 解像度: 1920x1080（1080p）
  - フレームレート: 24fps
  - 品質: 制限あり 15,000-20,000 Kb/s
- [ ] 各タイムラインをレンダーキューに追加
- [ ] 一括レンダリング実行
- [ ] 出力先: `~/footage/2026-03-16/export/`
